Email digests are one way to consume project results. Your project feed is still the source of truth in app, RSS/Atom, and CLI.

## Set up digest delivery

During project setup:

1. Add a project name.
2. Set maximum papers per email.
3. Add your email when prompted.
4. Save project.

## Change digest volume

To reduce inbox volume, edit **Max papers per email** in delivery settings.

If you want zero emails for a while, pause the project. This also pauses feed updates.

## What digest emails contain

A digest email includes a selected set of papers from your project, up to your max-paper limit.

Some days may include fewer papers.

## Common controls

- Edit project delivery settings
- Pause or resume project
- Tune keywords to reduce low-value matches
